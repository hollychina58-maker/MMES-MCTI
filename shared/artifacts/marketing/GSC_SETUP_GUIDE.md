# Google Search Console (GSC) 配置指南

本文档为 MEMS-MCTI 员工提供在 https://mmes-mcti.hollychina58.workers.dev 网站上设置 Google Search Console 的详细步骤。

---

## 1. Google Search Console 申请步骤

### 1.1 访问 GSC

1. 打开浏览器，访问 **https://search.google.com/search-console**
2. 使用您的 Google 账号登录（如未登录）

### 1.2 添加资源

1. 进入 GSC 后，点击左侧菜单 **"资源"**
2. 点击页面右上角的 **"+ 添加资源"** 按钮

   > [截图1: GSC添加资源按钮位置]

### 1.3 输入网站 URL

1. 在弹出的对话框中，选择资源类型：
   - **网域**：输入完整域名（如 `mmes-mcti.hollychina58.workers.dev`）
   - **网址前缀**：输入具体 URL（如 `https://mmes-mcti.hollychina58.workers.dev`）

2. 推荐选择 **"网址前缀"**，因为可以更灵活地使用不同的验证方法

3. 在输入框中输入：
   ```
   https://mmes-mcti.hollychina58.workers.dev
   ```

4. 点击 **"继续"** 按钮

   > [截图2: GSC添加资源 - 输入URL界面]

### 1.4 选择验证方法

GSC 提供多种验证方法，推荐使用 **HTML 文件验证**：

| 验证方法 | 难度 | 说明 |
|---------|------|------|
| HTML 文件 | 简单 | 下载HTML文件，上传到网站根目录 |
| HTML 标记 | 简单 | 在网站 `<head>` 中添加 meta 标签 |
| 域名提供商 | 中等 | 通过 DNS 配置验证 |
| Google Analytics | 简单 | 使用已有 GA 代码验证 |

**推荐步骤（HTML文件验证）：**

1. 选择 **"HTML 文件"** 验证方法
2. 点击 **"下载"** 按钮，下载验证文件

   > [截图3: GSC验证方法选择界面]

### 1.5 下载验证 HTML 文件

1. 浏览器将下载一个 HTML 文件，文件名格式为：
   ```
   google[hash].html
   ```
   例如：`google1234567890abcde.html`

2. 将此文件保存到本地计算机的指定位置（稍后需要上传到服务器）

   > [截图4: 下载的验证HTML文件]

## 2. 验证方法（HTML 文件）

### 2.1 上传验证文件到网站根目录

由于网站使用 Cloudflare Pages（Workers）托管，需要通过以下方式部署验证文件：

#### 方法一：通过 Cloudflare Dashboard 上传

1. 登录 **Cloudflare Dashboard** (https://dash.cloudflare.com)
2. 进入您的站点 **mmes-mcti.hollychina58.workers.dev**
3. 点击 **"Pages"** 选项卡
4. 选择您的项目，点击 **"部署"**
5. 在构建输出目录中添加验证文件

#### 方法二：通过 Git 上传（推荐）

1. 将验证文件复制到项目根目录
2. 提交到 Git 仓库：
   ```bash
   git add google[hash].html
   git commit -m "Add GSC verification file"
   git push
   ```
3. Cloudflare Pages 会自动部署新文件

### 2.2 验证文件位置要求

- 文件必须放在网站根目录
- 访问 URL 应为：
  ```
  https://mmes-mcti.hollychina58.workers.dev/google[hash].html
  ```

### 2.3 完成验证

1. 返回 Google Search Console
2. 点击 **"验证"** 按钮
3. 等待验证结果

**成功标志：** 页面显示 **"已验证 - 所有权已确认"**

   > [截图5: GSC验证成功界面]

### 2.4 验证失败排查

| 问题 | 解决方法 |
|------|---------|
| 文件 404 | 检查文件是否在根目录，文件名是否正确 |
| 部署延迟 | 等待几分钟后再试，Cloudflare Pages 需要时间部署 |
| 路径错误 | 确保 URL 完全匹配：`https://mmes-mcti.hollychina58.workers.dev/google[hash].html` |

## 3. Sitemap 提交

### 3.1 进入 Sitemap 提交页面

1. 在 GSC 左侧菜单中，点击 **"索引"**
2. 选择 **"Sitemaps"** 子菜单

   > [截图6: GSC Sitemaps菜单位置]

### 3.2 提交 sitemap

1. 在 "添加 sitemap" 输入框中，输入：
   ```
   sitemap.xml
   ```
   （完整 URL：`https://mmes-mcti.hollychina58.workers.dev/sitemap.xml`）

2. 点击 **"提交"** 按钮

   > [截图7: GSC提交sitemap界面]

### 3.3 查看 Sitemap 状态

提交后，GSC 会显示以下状态：

| 状态 | 含义 |
|------|------|
| 已成功 | Sitemap 已接收，等待爬取 |
| 已提交 | 已提交，等待处理 |
| 错误 | 检查 sitemap.xml 格式是否正确 |

**查看详情：** 点击 sitemap URL 可以查看：
- 发现的 URL 数量
- 索引数量
- 错误数量
- 最后爬取时间

   > [截图8: GSC Sitemap状态详情]

## 4. 初始检查清单

完成 GSC 配置后，建议进行以下初始检查：

### 4.1 查看索引覆盖报告

1. 进入 **"索引" > "覆盖"**
2. 查看报告摘要，特别关注：

   | 状态 | 说明 | 需要操作？ |
   |------|------|-----------|
   | 有效 | 页面已收录 | 无 |
   | 有效但有警告 | 页面收录但有问题 | 检查警告原因 |
   | 发现但未收录 | Google 发现但未索引 | 需要优化或提交 |
   | 已排除 | 故意不显示 | 检查是否正常 |

3. 如有 "发现但未收录" 的页面：
   - 检查是否是被 robots.txt 阻止
   - 检查是否 canonical 指向其他页面
   - 考虑使用 "请求索引" 功能

   > [截图9: GSC索引覆盖报告]

### 4.2 检查关键词报告

1. 进入 **"流量增长" > "关键词"**
2. 查看：
   - 用户搜索哪些词找到了您的网站
   - 点击率 (CTR)
   - 平均排名

3. 识别机会关键词（排名5-20位），这些页面有优化潜力

   > [截图10: GSC关键词报告]

### 4.3 检查爬取错误

1. 进入 **"设置" > "爬取"**
2. 查看 **"爬取错误"** 报告
3. 常见错误类型：

   | 错误类型 | 可能原因 | 解决方法 |
   |---------|---------|---------|
   | Not Found (404) | 页面已删除或URL错误 | 修复链接或设置301重定向 |
   | Server Error (5xx) | 服务器问题 | 检查服务器/Workers日志 |
   | Blocked by robots.txt | 被阻止抓取 | 检查 robots.txt 配置 |

   > [截图11: GSC爬取错误报告]

### 4.4 设置目标国家

1. 进入 **"设置" > "目标国家"**
2. 为网站选择目标国家/地区

   **建议配置：**

   | 国家 | 权重 | 说明 |
   |------|------|------|
   | 中国 | 高 | 主要市场 |
   | 美国 | 高 | 英文市场 |
   | 德国 | 中 | 欧洲市场 |
   | 俄罗斯 | 中 | 俄语市场 |

3. 保存设置

   > [截图12: GSC目标国家设置]

## 5. 定期检查任务（每月）

为保持 SEO 效果，建议每月执行以下检查任务：

### 5.1 检查索引覆盖

**操作步骤：**
1. 登录 GSC
2. 进入 **"索引" > "覆盖"**
3. 检查 "有效" 状态的比例
4. 如有新增错误，记录并处理

**关注指标：**
- 总索引页面数是否稳定
- 错误数量是否增加
- "发现但未收录" 是否有改善

### 5.2 查看 TOP 搜索词变化

**操作步骤：**
1. 进入 **"流量增长" > "关键词"**
2. 设置时间范围为 "过去28天" vs "之前的28天"
3. 比较：
   - 哪些关键词排名上升
   - 哪些关键词排名下降
   - 哪些新关键词带来流量

**关注：**
- 排名下降超过5位的关键词
- 点击率下降超过20%的关键词

### 5.3 检查爬取错误

**操作步骤：**
1. 进入 **"设置" > "爬取" > "爬取统计信息"**
2. 检查：
   - 平均响应时间
   - 每日爬取次数
   - 爬取错误数量

**错误处理优先级：**

| 优先级 | 错误类型 | 处理时限 |
|--------|---------|---------|
| 高 | 服务器错误 (5xx) | 24小时内 |
| 中 | 404 错误（重要页面） | 3天内 |
| 低 | 404 错误（已删除页面） | 每周批量处理 |

## 附录：常见问题 (FAQ)

### Q1: 验证失败怎么办？

1. 确认文件已正确上传到根目录
2. 清除浏览器缓存后重试
3. 尝试使用其他验证方法（如 HTML 标签验证）

### Q2: Sitemap 显示 "无法获取"？

1. 确认 sitemap.xml 可通过 URL 直接访问
2. 检查 robots.txt 是否允许爬取 sitemap
3. 验证 sitemap.xml 格式正确

### Q3: 页面显示 "发现但未收录"？

可能原因：
- 内容质量不足
- 被 robots.txt 阻止
- canonical 指向其他页面
- 网站较新，Google 还在评估

解决方法：
- 提升内容质量
- 检查 robots.txt
- 使用 "请求索引" 功能

### Q4: 如何移除已删除的页面？

1. 在 GSC 中搜索该 URL
2. 点击 "请求移除"
3. 选择 "暂时移除" 或 "永久移除"
